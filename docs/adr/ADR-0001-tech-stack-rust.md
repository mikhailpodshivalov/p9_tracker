# ADR-0001: Основной стек реализации

- Date: 2026-02-08
- Status: Accepted
- Context: старт проекта `p9_tracker`, этап 1 (планирование)

## Decision

Основной язык и платформа реализации: `Rust`.

## Rationale

- Снижение риска memory/concurrency багов для долгоживущего solo-кодбейса.
- Подходящая модель для разделения `UI`/`Scheduler`/`Audio RT` с контролем потоков.
- Соответствует зафиксированным рискам этапа 1 (realtime и потокобезопасность).

## Consequences

- В этапах реализации приоритет на Rust-экосистему для core-компонентов.
- Больше времени на вход в язык в начале, меньше технического долга по мере роста проекта.
- Архитектурные интерфейсы и контракты v0 фиксируются с учётом Rust-модульности.

## Out of Scope

- Выбор конкретных crates для каждого компонента.
- Реализация и бенчмарки.
